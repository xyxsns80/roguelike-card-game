## 1. 概述

### 1.1 系统简介

抽卡系统是玩家获取角色卡牌的主要途径。玩家消耗游戏内货币，从指定卡池中随机获取角色或道具。

**一句话描述：** 用货币换卡牌，有保底不亏。

### 1.2 设计目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| **养成深度** | 提供长期追求目标，驱动玩家持续游玩 | P0 |
| **收入来源** | 核心付费点，支撑游戏运营 | P0 |
| **公平感知** | 保底机制确保付出必有回报 | P0 |
| **惊喜体验** | 随机性带来情绪波动和社交话题 | P1 |
| **操作便捷** | 3步完成抽卡，流程简单流畅 | P1 |

### 1.3 适用范围

- 游戏版本：v1.0 起生效
- 平台：iOS / Android / Web
- 用户：所有玩家

### 1.4 相关系统

| 系统名称 | 关系 | 接口说明 |
|----------|------|----------|
| 货币系统 | 前置 | 查询/扣除货币 |
| 角色系统 | 后置 | 新角色入库、重复转化 |
| 背包系统 | 后置 | 道具入库 |
| 引导系统 | 并行 | 新手抽卡教学 |
| 充值系统 | 关联 | 货币不足时跳转 |

## 2. 设计意图

### 2.1 为什么需要抽卡系统？

**核心价值：** 抽卡是连接"付费"与"养成"的桥梁。

```
付费 → 货币 → 抽卡 → 角色/道具 → 养成 → 战力提升
```

### 2.2 解决什么问题？

| 问题 | 解决方案 |
|------|----------|
| 玩家想要特定角色但无法保证获得 | 提供「保底机制」，90抽必得SSR |
| 玩家担心"氪金无底洞" | 明确概率公示，180抽必得UP角色 |
| 新玩家不知道抽什么 | 提供「新手池」，优惠价格体验抽卡 |
| 老玩家追求特定角色 | 提供「限定池」「UP池」，提高目标角色概率 |

### 2.3 期望的玩家体验

**情绪曲线：**

```
期待 ──────────── 抽卡前
    ╱
   ╱
兴奋 ──────────── 动画播放
  ╱
 ╱
惊喜 ──────────── 出SSR（高潮）
      ╲
       ╲
满足 ──── 看到结果，角色入队
```

**关键体验点：**
1. 抽卡前：期待感、选择困难（选哪个池）
2. 抽卡中：紧张感、视觉冲击
3. 抽卡后：满足感或再来一次的冲动

### 2.4 参考案例

| 游戏 | 可借鉴点 | 备注 |
|------|----------|------|
| 原神 | 90抽小保底、180抽大保底 | 业界标准，玩家认知度高 |
| 明日方舟 | 公招系统（定向筛选） | 可作为未来扩展方向 |
| 崩坏：星穹铁道 | 十连动画分批展示 | 视觉体验好 |
| 公主连结 | 重复角色转化为碎片 | 解决重复产出问题 |

### 3.3 货币类型

| 货币名称 | 图标 | 获取方式 | 消耗场景 |
|----------|------|----------|----------|
| **原石** | 💎 | 充值、任务、活动 | 兑换抽卡货币 |
| **相遇之缘** | 🎫 | 原石兑换、活动 | 常驻池、UP池抽卡 |
| **纠缠之缘** | 🎟️ | 原石兑换、活动 | 限定池抽卡 |
| **新手券** | 🎫 | 新手礼包 | 新手池（仅限20抽） |

**兑换比例：**
- 160 原石 = 1 相遇之缘
- 160 原石 = 1 纠缠之缘
- 新手券免费发放

### 3.4 概率机制

**基础概率（所有卡池通用）：**

| 稀有度 | 概率 | 说明 |
|--------|------|------|
| **SSR** | 1.600% | 5星角色 |
| **SR** | 13.000% | 4星角色/武器 |
| **R** | 85.400% | 3星武器 |

**概率提升机制（软保底）：**

从第 **74 抽** 开始，每抽 SSR 概率提升 **6%**：

| 抽数 | SSR概率 |
|------|---------|
| 1-73 | 1.600% |
| 74 | 7.600% |
| 75 | 13.600% |
| 76 | 19.600% |
| ... | ... |
| 89 | 77.600% |
| 90 | **100%**（硬保底） |

### 3.5 保底机制

#### 3.5.1 SSR保底（小保底）

```
规则：90抽内必出SSR
触发：获得任意SSR后重置计数
继承：同类型卡池保底计数继承
```

#### 3.5.2 UP保底（大保底）

```
规则：180抽内必得当期UP角色
触发：获得UP角色后重置计数
继承：同UP角色池继承，更换UP角色后重置
```

#### 3.5.3 SR保底

```
规则：10抽内必出SR或以上
触发：获得SR/SSR后重置计数
```

#### 3.5.4 保底继承规则表

| 从池 | 到池 | SSR保底 | UP保底 |
|------|------|---------|--------|
| 常驻池 | 常驻池 | ✅ 继承 | ❌ 无 |
| UP池A | UP池A | ✅ 继承 | ✅ 继承 |
| UP池A | UP池B | ✅ 继承 | ❌ 重置 |
| 限定池A | 限定池A | ✅ 继承 | ✅ 继承 |
| 限定池A | 限定池B | ✅ 继承 | ❌ 重置 |
| 常驻池 | UP池 | ✅ 继承 | ❌ 无 |

| 稀有度 | 类型 | 单独概率 | 说明 |
|--------|------|----------|------|

| SR | 角色 | 13.000% | - |
| R | 武器 | 85.400% | - |

### 4.2 保底计数器规则

#### 4.2.1 计数器数据结构

```typescript
interface PityCounter {
  // SSR保底计数（0-89）
  ssrPity: number;
  
  // UP保底计数（0-179）
  upPity: number;
  
  // 上次SSR是否为UP（大保底标记）
  lastSsrWasUp: boolean;
  
  // SR保底计数（0-9）
  srPity: number;
  
  // 卡池类型
  poolType: 'permanent' | 'up' | 'limited' | 'beginner';
  
  // UP角色ID（仅UP池）
  upCharacterId?: string;
}
```

#### 4.2.2 计数器更新逻辑

```
抽卡结果判定流程：

1. 检查 SSR 保底
   IF ssrPity >= 89 THEN
     必出 SSR
     ssrPity = 0
     执行 UP 判定
   ELSE
     计算实际概率 = 基础概率 + 软保底加成
     按概率判定是否出 SSR
     IF 出 SSR THEN
       ssrPity = 0
       执行 UP 判定
     ELSE
       ssrPity++
   
2. 检查 SR 保底（仅当未出SSR时）
   IF srPity >= 9 THEN
     必出 SR
     srPity = 0
   ELSE
     按概率判定
     IF 出 SR THEN srPity = 0
     ELSE srPity++
   
3. UP 判定（仅当出SSR时）
   IF poolType == 'up' THEN
     IF lastSsrWasUp == true THEN
       必出 UP 角色
       lastSsrWasUp = false
       upPity = 0
     ELSE
       50% 概率出 UP
       IF 出 UP THEN
         lastSsrWasUp = false
         upPity = 0
       ELSE
         lastSsrWasUp = true
         upPity++
```

### 4.4 产出价值评估

#### 4.4.1 期望价值计算

```
单抽期望价值 = Σ(产出概率 × 产出价值)

SSR 价值：10000
SR 价值：1000
R 价值：10

期望价值 = (0.016 × 10000) + (0.13 × 1000) + (0.854 × 10)
        = 160 + 130 + 8.54
        = 298.54

约等于 300 价值单位/抽
```

#### 4.4.2 保底价值下限

```
90抽保底 SSR 价值 = 10000
单抽成本 = 160 原石
90抽成本 = 14400 原石

最低价值/成本比 = 10000 / 14400 ≈ 0.69
```

#### 5.3.3 抽卡动画流程

```

2. 屏幕变暗 → 黑色遮罩淡入 (0.2s)
3. 粒子汇聚 → 从四周向中心汇聚 (0.5s)
```

### 6.3 业务逻辑伪代码

#### 6.3.1 抽卡主流程

```python
def gacha_pull(player_id, pool_id, pull_count):
    # 1. 参数校验
    if pull_count not in [1, 10]:
        return error("Invalid pull count")
    
    # 2. 获取卡池配置
    pool_config = get_pool_config(pool_id)
    if not pool_config:
        return error("Pool not found")
    
    if not is_pool_open(pool_config):
        return error("Pool is closed")
    
    # 3. 检查新手池限制
    if pool_config.type == POOL_TYPE_BEGINNER:
        beginner_count = get_beginner_pull_count(player_id)
        if beginner_count >= 20:
            return error("Beginner pool limit reached")
        if beginner_count + pull_count > 20:
            pull_count = 20 - beginner_count  # 自动调整为剩余次数
    
    # 4. 计算货币消耗
    cost = pool_config.ten_cost if pull_count == 10 else pool_config.single_cost
    
    # 5. 检查货币是否足够
    if not has_enough_currency(player_id, pool_config.currency_type, cost):
        return error("Insufficient currency")
    
    # 6. 扣除货币（事务开始）
    with transaction():
        deduct_currency(player_id, pool_config.currency_type, cost)
        
        # 7. 获取保底计数器
        pity = get_pity_counter(player_id, pool_config.type)
        
        # 8. 执行抽卡
        rewards = []
        for i in range(pull_count):
            reward = single_pull(pool_config, pity)
            rewards.append(reward)
            
            # 更新保底计数
            update_pity_counter(pity, reward.rarity, reward.is_up)
        
        # 9. 保存保底计数器
        save_pity_counter(player_id, pity)
        
        # 10. 发放奖励
        for reward in rewards:
            grant_reward(player_id, reward)
        
        # 11. 更新新手池计数
        if pool_config.type == POOL_TYPE_BEGINNER:
            increment_beginner_count(player_id, pull_count)
    
    # 12. 返回结果
    return success({
        "cost": {"currencyType": pool_config.currency_type, "amount": cost},
        "rewards": rewards,
        "pityCounter": pity,
        "remainingCurrency": get_currency_balance(player_id, pool_config.currency_type)
    })


def single_pull(pool_config, pity):
    """单次抽卡逻辑"""
    # 1. 判定SSR
    ssr_rate = calculate_ssr_rate(pity.ssr_pity)
    if pity.ssr_pity >= 89 or random() < ssr_rate:
        # 出SSR
        pity.ssr_pity = 0
        
        # UP判定
        if pool_config.pool_type == POOL_TYPE_UP:
            if pity.last_ssr_was_up or random() < 0.5:
                # 出UP角色
                pity.last_ssr_was_up = False
                pity.up_pity = 0
                return get_up_character(pool_config)
            else:
                # 出非UP角色
                pity.last_ssr_was_up = True
                pity.up_pity += 1
                return get_non_up_ssr(pool_config)
        else:
            return get_random_ssr(pool_config)
    
    # 2. 判定SR
    pity.ssr_pity += 1
    if pity.sr_pity >= 9 or random() < pool_config.sr_rate:
        pity.sr_pity = 0
        return get_random_sr(pool_config)
    
    # 3. 出R
    pity.sr_pity += 1
    return get_random_r(pool_config)


def calculate_ssr_rate(ssr_pity):
    """计算软保底概率"""
    BASE_RATE = 0.016  # 1.6%
    SOFT_PITY_START = 74
    SOFT_PITY_INCREMENT = 0.06  # 6%
    
    if ssr_pity < SOFT_PITY_START:
        return BASE_RATE
    else:
        return BASE_RATE + (ssr_pity - SOFT_PITY_START + 1) * SOFT_PITY_INCREMENT
```

#### 6.5.1 请求签名算法

```python
def generate_sign(player_id, pool_id, pull_count, timestamp, secret_key):
    """生成请求签名"""
    params = f"{player_id}|{pool_id}|{pull_count}|{timestamp}"
    return hmac_sha256(secret_key, params)


def verify_sign(request, secret_key):
    """验证请求签名"""
    expected_sign = generate_sign(
        request.player_id,
        request.pool_id,
        request.pull_count,
        request.timestamp,
        secret_key
    )
    return request.sign == expected_sign
```

#### 6.5.2 防重放攻击

```python
def check_replay_attack(timestamp, player_id):
    """检查重放攻击"""
    # 1. 时间戳校验（5分钟内有效）
    if abs(time.now() - timestamp) > 300:
        return False
    
    # 2. 检查是否已处理过该请求（Redis去重）
    cache_key = f"gacha:req:{player_id}:{timestamp}"
    if redis.exists(cache_key):
        return False
    
    # 3. 标记请求已处理（5分钟过期）
    redis.setex(cache_key, 300, "1")
    return True
```
